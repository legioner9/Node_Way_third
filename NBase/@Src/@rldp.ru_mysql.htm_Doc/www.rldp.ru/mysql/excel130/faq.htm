Вопросы и ответы

9.1: Когда я буду использовать Excel, чтобы отредактировать данные из живой базы данных MySQL, мои изменения перепишут изменения, внесенные другими пользователями? Например, возможно они использовали MySQL Workbench, чтобы отредактировать те же самые данные в то же время.

Функция optimistic updates проверяет внешние правки и уведомляет вас относительно их существования прежде, чем передать любые изменения. Если внешние правки обнаружен, можно тогда выбрать, переписать ли их изменения. Этот выбор позволен по умолчанию и может быть отключен (чтобы использовать пессимистические обновления). Отключение этой опции означает, что внешние изменения будут всегда переписываться. Другими словами, выбор ваш.

9.2: Я установил MySQL for Excel, но не могу найти его в Microsoft Excel. Как его запустить?

MySQL for Excel автоматически добавляется к меню данных Microsoft Excel. Ищите MySQL for Excel, по умолчанию это будет перечислено на правой стороне главного меню.

Если этого там нет, то вам, возможно, придется повторно установить плагин. Но прежде, чем сделать это, сначала проверьте, перечисляется ли он в списке "Add/Remove Programs" в Microsoft Windows. В противном случае, это не было установлено. Затем проверьте список Excel Add-Ins. Для Office 2007 это найдено, щелкнув по лого Офиса в Excel (левый верхний угол), нажмите Excel Options, затем выберите Add-Ins. Перечислен ли MySQL for Excel в COM Add-in? Если это так, рассмотрите регистрацию отчета об ошибках (bugs.mysql.com) или попытайтесь повторно установить плагин.

9.3: Я жму на Edit Data и после импортирования данных в Excel я не могу сортировать или переместить колонки. Как я могу сделать это?

Чтобы поддержать отображение строк и колонок в рабочем листе Excel в соответствии с MySQL, никакое изменение не разрешено на рабочем листе (т.е. сортировка и удаление). Если необходимо изменить данные там, можно сделать это, щелкнув правой кнопкой мыши в окне Edit Data и выбрав Exit Edit Mode.

9.4: Редактируя данные таблицы MySQL, рабочий лист Excel где данные лежат, защищен. Как можно не защитить его?

Рабочий лист Excel защищен, чтобы не позволить изменения порядка строк и колонок. Пароль, используемый для защиты, является GUID, созданным во время выполнения так, чтобы защита не была нарушена ни в каком случае. Если вы хотите не защитить рабочий лист, чтобы управлять вашими данными, можно сделать это, щелкнув правой кнопкой мыши по окну Edit Data и выбрав Exit Edit Mode.
