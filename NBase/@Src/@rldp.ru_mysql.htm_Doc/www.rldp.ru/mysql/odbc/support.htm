Глава 9. Поддержка Connector/ODBC

Есть много различных мест, где можно получить поддержку использования Connector/ODBC. Всегда пробуйте Connector/ODBC Mailing List или Connector/ODBC Forum. См. раздел 9.1 для помощи прежде, чем сообщить об определенной ошибке или проблеме.

9.1. Поддержка сообществом Connector/ODBC

Общественная поддержка от опытных пользователей также доступна через ODBC Forum. Можно также найти помощь от других пользователей на других форумах MySQL, расположенных на http://forums.mysql.com. См. MySQL Community Support at the MySQL Forums.

9.2. Как сообщить о проблемах или ошибках Connector/ODBC

Если вы сталкиваетесь с трудностями или проблемами с Connector/ODBC, начните, делая файл журнала из ODBC Manager и Connector/ODBC. Процедура как сделать это описана в разделе 5.8.

Проверьте файл трассировки Connector/ODBC, чтобы узнать то, что могло быть неправильным. Определите, какие запросы были сделаны, ища последовательность >mysql_real_query в файле myodbc.log.

Кроме того, попытайтесь делать запросы из клиента mysql или из admndemo. Это помогает вам определить, где ошибка: в Connector/ODBC или в MySQL.

Идеально, включайте следующую информацию в свой отчет об ошибках:

  • ОС и ее версия.

  • Версия Connector/ODBC.

  • Тип и версия ODBC Driver Manager.

  • Версия сервера MySQL.

  • Трассировка ODBC из Driver Manager.

  • Файл журнала Connector/ODBC из Connector/ODBC.

  • Простой восстанавливаемый образец.

Чем больше информации вы предоставляете, тем более вероятно, что мы сможем решить проблему.

Если вы неспособны узнать что не так, последний выбор состоит в том, чтобы создать архив в формате tar или zip, который содержит файл трассировки Connector/ODBC, файл журнала ODBC и файл README, который объясняет проблему. Начните отчет об ошибках для нашей базы данных ошибок на http://bugs.mysql.com/, затем щелкните по вкладке Files в отчете об ошибках для получения инструкций относительно загрузки архива в базу данных ошибок. Только у инженеров MySQL есть доступ к файлам, которые вы загружаете, и мы очень осторожны с данными.

Если можно создать программу, которая также демонстрирует проблему, пожалуйста, включайте ее в архив также.

Если программа работает с другим SQL-сервером, включайте файл журнала ODBC, где вы выполняете точно те же самые SQL-операторы так, чтобы мы могли сравнить результаты между этими двумя системами.

Помните что, чем больше информации можно поставлять нам, тем более вероятно, что мы сможем решить проблему.
