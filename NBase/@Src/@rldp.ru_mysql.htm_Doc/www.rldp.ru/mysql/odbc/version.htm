Глава 2. Версии Connector/ODBC

Это версии Connector/ODBC, которые в настоящее время доступны:

  • Connector/ODBC 8.0: добавляет поддержку MySQL Server 8.0, включая caching_sha2_password и связанный признак соединния GET_SERVER_PUBLIC_KEY. Для дополнительных деталей см. Connector/ODBC 8.0 release notes.

  • Connector/ODBC 5.3: подходит для Версий сервера MySQL между 4.1 и 5.7. Это не работает с 4.0 или более ранними выпусками и не поддерживает все функции MySQL 8. Это соответствует спецификации ODBC 3.8 и содержит ключевые особенности ODBC 3.8, включая самоидентификацию как драйвер ODBC 3.8, передачу параметров вывода (поддержано только для двоичных типов) и поддержку признака связи SQL_ATTR_RESET_CONNECTION (только для драйвера Unicode). Connector/ODBC 5.3 также вводит GTK+-based библиотеку установки, предоставляя диалог установки GUI DSN на некоторых основанных на Unix системах. Библиотека в настоящее время включается в Oracle Linux 6 и Debian 6. Другие новые особенности в 5.3 включают файл DSN и закладки, см. release notes for the 5.3 series.

    Connector/ODBC 5.3.11 добавил поддержку caching_sha2_password, добавив признак связи GET_SERVER_PUBLIC_KEY.

  • Connector/ODBC 5.2: модернизирует ANSI-драйер Connector/ODBC 3.51 до базы кода 5.x. Это также включает новые особенности, такие как предоставление возможности подготовки запросов на серверной стороне. Во время установки можно выбрать драйвер Unicode для самой широкой совместимости с источниками данных, используя различные наборы символов или ANSI для оптимальной работы с более ограниченным диапазоном наборов символов. Это работает с версиями MySQL 4.1.1 и выше.

  • Connector/ODBC 5.1: частично переписанный код 3.51, чтобы работать с версиями MySQL 4.1.1 и более новыми.

    Connector/ODBC 5.1: также включает следующие изменения и улучшения по сравнению с этими выпусками 3.51:

    • Улучшенная поддержка на платформах Windows 64.

    • Полная поддержка Unicode на уровне драйвера. Это включает поддержку типа данных SQL_WCHAR и логинов в Unicode, паролей и конфигураций DSN. См. Microsoft Knowledgebase Article #716246.

    • Поддержка типа данных SQL_NUMERIC_STRUCT, который обеспечивает более легкий доступ к точному определению числовых значений. Для получения дополнительной информации посмотрите Microsoft Knowledgebase Article #714556.

    • Родная библиотека программы установки Windows. Это заменяет Qt-библиотеку интерфейса для формирования информации о DSN в применении источников данных ODBC.

    • Поддержка дескриптора ODBC, который улучшает обработку и метаданные данных о параметрах и столбцах. См. Microsoft Knowledgebase Article #716339.

  • Connector/ODBC 3.51, также известен как MySQL ODBC 3.51 driver, это 32-bit драйвер ODBC. У Connector/ODBC 3.51 есть поддержка ODBC 3.5x level 1 (complete core API + level 2 features), чтобы продолжить обеспечивать всю функциональность ODBC для доступа к MySQL.

Руководство для версий Connector/ODBC, более старого, чем 5.3, может быть найдено в соответствующем пакете.

Версии Connector/ODBC ранее, чем 3.51 не были полностью совместимы со спецификацией ODBC.

Номера версий для продуктов MySQL отформатированы как X.X.X. Однако инструменты Windows (Панель управления, отображение свойств) могут показать номера версий как XX.XX.XX. Например, номер версии 5.0.9 официального MySQL может быть показан инструментами Windows как 5.00.09. Эти две версии то же самое, только форматы отображения числа отличаются.
