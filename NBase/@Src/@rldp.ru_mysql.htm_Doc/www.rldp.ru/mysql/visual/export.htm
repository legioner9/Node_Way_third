Глава 9. MySQL Data Export Tool

У MySQL for Visual Studio есть экспортный инструмент данных, который создает файл дампа для базы данных MySQL.

Рис. 9.1. MySQL for Visual Studio Data Export Tool: главное окно


Создание дампа существующей базы данных MySQL

Чтобы открыть новое окно для MySQL Data Export tool, пользователь должен создать новую связь, используя Server Explorer Window в Visual Studio. Как только связь устанавливается, контекстное меню может быть открыто, щелкнув правой кнопкой мыши по узлу подключения. Из меню выберите MySQL Data Export . Новое окно открывается для текущей связи. Пользователь может выбрать одну или более баз данных, чтобы включить в дамп.

Рис. 9.2. MySQL for Visual Studio Data Export Tool: контекстное меню связи


Выполните эти шаги, чтобы создать дамп для баз данных MySQL:

  1. Выберите все базы данных и их объекты, которые будут включены в дамп.

    Рис. 9.3. MySQL for Visual Studio Data Export Tool: выбор базы данных


  2. Очень важно выбрать желаемые параметры настройки для дампа: будет ли он включать данные, войдут ли операции по вставке в расширенный способ и так далее. В главном окне MySQL Database Export показываются основные варианты для дампа. Кроме того, нажимая на кнопку Advanced, более определенные варианты могут также быть выбраны.

    Рис. 9.4. MySQL for Visual Studio Data Export Tool: расширенные настройки


  3. Когда выбор вариантов сделан, дайте имя файлу результата, который будет создан. Если никакой путь не дан для файла результата, путь по умолчанию, который будет использоваться, My Documents под папкой пользователя.

    Рис. 9.5. MySQL for Visual Studio Data Export Tool: создание файла дампа


  4. Фильтр может быть применен на список схем для отобранной связи. С ним пользователь может легко определить местонахождение баз данных, которые будут включены в дамп.

    Рис. 9.6. MySQL for Visual Studio Data Export Tool: фильтрация схем


  5. После отбора вариантов и названия файла дампа пользователь может нажать кнопку Export, которая производит дамп.

    Рис. 9.7. MySQL for Visual Studio Data Export Tool: просмотр произведенного скрипта


У каждого дампа могут быть различные параметры настройки. После формирования операции по дампу настройки могут быть сохранены в файл для более позднего использования. Этот файл включает: отобранная связь, название файла и база данных или базы данных и объекты для дампа. Расширение файла для файла настроек .dumps.

Рис. 9.8. MySQL for Visual Studio Data Export Tool: сохранить файл настроек


Сохраненный файл может быть загружен в MySQL Data Export нажав кнопку Load Settings.

Рис. 9.9. MySQL for Visual Studio Data Export Tool: открытие файла настроек
